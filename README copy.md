## Разработка

Для запуска приложения в режиме разработки:
1. Скопировать `.env.example` в `.env`
2. Установить зависимости `yarn install`
3. Запустить сервер `yarn dev`

## Обновление зависимостей

### Базовый образ
1. Перейти в папку с базовыи докер файлом 
```
cd docker_base
```
2. Запустить команду сборки образа с нужной версией
```
./build.sh 1.0.0
```

## Стиль написания кода
1. Название папок пишем в cebab-case - ```actor-casting-reply```
2. Название файлов компонентов и выделенных типов в UpperCamelCase - ```CastingCard.tsx, Error.ts```
3. Название остальных файлов в camelСase - ```loadingHandler.ts```
4. Название переменных констант в UPPER_SNAKE_CASE - ```AUTH_ROUTES```
5. Название вложенных констант в snake_case - ```AUTH_ROUTES.sign_up```
6. Роуты в cabab-case - ```auth/sign-in-form```
7. Для обеспечения согласованности и читаемости, мы используем следующую схему именования ключей в конфигурации маршрутов:

   index: Обозначает индексный маршрут (главная страница раздела). Например, /events — это индексный маршрут для раздела событий.
   show: Обозначает маршрут для отображения конкретного элемента. Например, /events/id — отображает информацию о событии с id. В случае вложенных маршрутов (например, проект внутри события), show используется для каждого уровня вложенности.
